# セキュリティ・プライバシーリスク一覧と対策

未成年者の利用を想定し、安全側に倒した設計とする。

| # | リスク | 影響 | 対策(実装) |
|---|--------|------|-----------|
| 1 | 鮮明な顔・部屋の映像が他者に見える | プライバシー侵害・特定 | 送信前にCanvasで映像自体へ強ぼかし+縮小。生トラックはPeerConnectionに追加しない。CSS表示だけのぼかしは不採用 |
| 2 | 音声の盗聴・生活音の漏洩 | プライバシー侵害 | `getUserMedia`で音声を要求しない。Permissions-Policyで`microphone=()`を全面無効化 |
| 3 | 映像のサーバー保存・録画 | 重大な情報漏洩 | P2P構成のため映像はサーバーへ到達しない。録画機能は作らない。管理者にも閲覧機能を作らない |
| 4 | 他人の学習履歴・個人情報の閲覧 | 個人情報漏洩 | 全テーブルにRLS。本人・管理者以外は行にアクセス不可。同室者に公開されるのは表示名・学習内容のみ(専用ビュー経由) |
| 5 | XP・実績の不正加算 | ランキング等の信頼低下 | XP/バッジはクライアントから書き込み不可(RLSでINSERT/UPDATE拒否)。`finish_session`等のSECURITY DEFINER RPCのみが付与し、サーバー時刻で検証 |
| 6 | なりすまし・不適切な映像・迷惑行為 | 未成年者への危害 | 通報(5分類)→管理者確認→利用停止。ブロックで同室回避。利用停止はDBフラグ+入室RPCで拒否 |
| 7 | 管理画面への不正アクセス | 全ユーザー情報漏洩 | `profiles.role`+`is_admin()`(SECURITY DEFINER)をRLSとサーバーレイアウトの両方で検証。管理操作は`admin_audit_logs`へ記録 |
| 8 | XSS | セッション乗っ取り | Reactの自動エスケープ。ユーザー入力に対する`dangerouslySetInnerHTML`不使用(使用箇所は静的なテーマ初期化スクリプト1点のみ)。ユーザー入力はZodで検証 |
| 9 | SQLインジェクション | データ破壊 | supabase-jsのパラメータ化クエリ/RPCのみ使用。文字列連結SQLなし |
| 10 | CSRF | 意図しない操作 | Supabase AuthのトークンはAuthorizationヘッダで送信(Cookie依存の状態変更なし)。Route HandlerはOrigin検証。SameSite=Lax |
| 11 | 通報・招待の濫用(スパム) | 迷惑メール送信 | レート制限(DB側: 招待は1日5件、通報は1日10件をRPCで制限) |
| 12 | 応援者への意図しない共有 | プライバシー侵害 | 本人の明示登録+応援者側の同意(トークン付きリンク)の二重同意。いつでも停止・削除可能。映像は共有しない |
| 13 | 第三者による勝手な応援者登録 | 監視の悪用 | 招待メールに同意リンク、同意するまで通知は送られない。本人にも一覧で可視化 |
| 14 | 決済情報の漏洩 | 金銭被害 | カード情報はStripeのみが保持(Checkout/Portal利用)。自社DBはStripe顧客ID・購読状態のみ |
| 15 | IP・端末情報の過剰収集 | プライバシー | 通報処理に必要な最小限(通報対象のセッションID)のみ。IPアドレスの独自保存はしない |
| 16 | シグナリングチャネルへの不正参加 | 盗聴・妨害 | Supabase Realtime のprivateチャネル+RLS(参加者のみJOIN可)。ルームIDはUUIDで推測不能 |
| 17 | 多重入室・終了済みセッション再入室 | 記録の不正 | activeセッションの部分ユニーク制約+`join_room`RPCで検証 |
| 18 | 退会後のデータ残存 | 忘れられる権利 | 退会時にauth.users削除→CASCADE+個人情報の匿名化。学習統計は匿名化された形でのみ残す |

## 残存リスク(既知・要運用対応)

- P2Pのため参加者間でIPアドレスが相互に露出しうる(WebRTCの一般的性質)。本番ではTURN経由を強制する設定(`iceTransportPolicy: 'relay'`)への切替オプションを用意。プライバシーポリシーに明記。
- ぼかしはクライアント側で適用されるため、悪意あるユーザーが改造クライアントで鮮明な映像を送る可能性はゼロではない → 受信側での追加ぼかし表示(二重化)+通報機能で緩和。
- 無料メール確認を無効化した環境では捨てメールでの大量登録が可能 → 本番はメール確認必須+レート制限。
