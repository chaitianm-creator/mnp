# 🏢 AI Virtual Office — AI社員が働く仮想オフィス

AI社員19名が並列で働く様子を可視化するバーチャルオフィスのデモアプリです。
**ビルド不要・依存ゼロの単一HTMLファイル**（`index.html`）で動きます。

「普通なら5〜10人必要な作業を、AIエージェントが並列でこなし、しかも数字（件数）が見える」
というコンセプトをそのまま画面にしたものです。

## 起動方法

```bash
# ブラウザで直接開くだけ
open ai-office/index.html

# またはローカルサーバーで
npx serve ai-office
```

Vercel / Netlify / GitHub Pages などに静的ファイルとしてそのまま公開できます。

## 画面の機能

| 機能 | 内容 |
|---|---|
| 🏢 バーチャルオフィス | 4部署（受付・営業フロア・バックオフィス・Web制作スタジオ）のフロアマップ。AI社員が席で作業し、吹き出しで現在のタスクを表示。コーヒーブレイクで歩き回り、来客が入口から受付に歩いてくる |
| 📊 KPIカウンター | **リスト獲得 / フォーム送信 / メール送信 / 受付対応 / アポイント** をリアルタイム集計（本日分＋累計） |
| 📡 活動ログ | 全社員のアクションを時系列でストリーム表示 |
| 👥 社員詳細 | 社員をクリックすると本日アクション数・架電数・トークン消費・概算コスト（円）を表示 |
| 🖥 制作案件 | HP制作パイプライン（要件定義→ワイヤー→デザイン→コーディング→ライティング→レビュー→公開）を案件ごとに進捗バー表示。公開数・成約数もカウント |
| 💴 経理 | 経理AIが全社員の利用トークンを集計し円換算。**Excel帳簿（CSV）ダウンロード**付き |
| ⏯ 操作 | 一時停止／速度 ×1・×3・×10（1秒=1分）。18:00になると日報を出して翌営業日へ |

## いまはシミュレーション、次は本物のエージェントへ

このデモは乱数ベースのシミュレーションですが、画面側は「イベントを受け取ってKPI・ログ・帳簿を更新する」だけの作りなので、裏側を本物のAIエージェントに差し替えられます。

想定アーキテクチャ:

```
[Claude Agent SDK / Claude API]
  ├─ リサーチAI  → Web検索・スクレイピングでリスト生成
  ├─ フォームAI  → Playwright等でフォーム自動入力
  ├─ メールAI    → Resend / Gmail API で送信・開封トラッキング
  ├─ 受付AI      → チャット/電話(音声API)の一次対応
  ├─ 経理AI      → 各エージェントのusage(トークン)をAPIレスポンスから集計
  └─ 制作チーム  → コード生成・デプロイ(Vercel API)
        ↓ イベント(JSON)をWebSocket / DBに書き込み
[このオフィス画面] … イベントを購読してアバター・KPI・帳簿を更新
```

置き換えのポイント:
- `fireJob()` が発火している箇所を、実エージェントからのイベント受信（WebSocket / Supabase Realtime など）に差し替える
- トークン数は各API呼び出しのレスポンス `usage.input_tokens / output_tokens` の実測値を使う（経理AIの帳簿がそのまま実コスト管理になる）
- 営業系の実行は必ず特定電子メール法・迷惑メール防止の観点でオプトイン/送信制限を設計すること

## 注意

- 数値・コストはすべてデモ用のシミュレーション値です（単価想定: 入力$3/100万tok・出力$15/100万tok、$1=155円）
- 外部通信は一切行いません（完全オフラインで動作）
